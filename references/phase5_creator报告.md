## Q1 激活触发评审

### 当前触发覆盖
description 字段已列触发词：「丁元英」「天道」「强势文化」「文化属性」「得救之道」「用丁元英的视角」「丁元英会怎么看」「文化属性论」「逆袭思维」「不被收割」+ 三句口语化变体（「帮我用丁元英的角度想想」「这事强势文化怎么看」「切到丁元英」）。

**评价**：人名 / 概念 / 命令式调用 三类都覆盖了，比一般 Skill 描述更细。但**真实使用场景词覆盖严重不足**。

### 漏掉的关键场景触发词

用户不会先想到"丁元英"再来调用 Skill——大多数情况下，用户是带着**情境痛点**来的，期望系统主动建议或匹配。以下高频痛点场景在 description 里**完全没出现**：

| 痛点场景 | 用户原话样例 | 当前能否触发 |
|---|---|---|
| 中年危机/职业焦虑 | 「35 岁被裁了怎么办」「中年危机」「我是不是废了」 | 不能 |
| 被收割后的复盘 | 「我被一个项目割了 30 万」「被骗了」「踩坑了」 | 部分（"不被收割"接近但用户说"被割了"不一定命中） |
| 创业/项目决策 | 「我想做个 XX 项目」「这个生意能不能干」「要不要 all in」 | 不能 |
| 暴富 / 救主叙事识别 | 「这个机会怎么样」「XX 一年赚 1000 万靠不靠谱」「跟着大佬抄作业」 | 不能 |
| 家庭关系/孝道困境 | 「家里要钱」「父母逼婚」「兄弟姐妹拎不清」 | 不能（但 SKILL.md 内附加规则 11-13 明确处理这类） |
| 失败/挫折后想找认知工具 | 「想透彻一点」「不想被情绪带着走」「想活明白」 | 不能 |
| 强势/弱势文化关键词的口语变体 | 「靠不上」「想自救」「不想躺平」「想逆袭但不蠢」 | 仅"逆袭思维"勉强命中 |

### 误触发风险

**几乎不存在**——「丁元英」「天道」是低歧义专名，普通用户不会无意触发。但反过来，正因为触发词过于"专名化"，**未读过原著的用户根本不知道有这个 Skill 可用**——这是更严重的问题。覆盖盲区 >> 误触发风险。

## Q2 角色扮演规则评审

### 问题路由是否清晰

**不清晰，是当前 SKILL.md 的最大执行短板。**

- 「角色扮演规则」section（L20-33）只规定了**风格层**（用「我」、不说"如果"、不跳出角色），**没有规定问题路由**：
  - 用户问"看人" → 该调模型 1（文化属性）+ 模型 2（自性自救）
  - 用户问"做生意" → 该调模型 3（戒贪三问）+ 模型 5（设计性强势警示）
  - 用户问"被打击了/想报复" → 该调模型 4（觉者由心生律，戒嗔）
  - 用户问"是不是该退出" → 该调模型 6（静守与退场）
- 「回答工作流」（L36-102）有 Step 1 问题分类（事实 / 框架 / 混合），**但没有"框架问题再分子类→落到哪个心智模型"的二级路由**。

结果：模型 LLM 必须在 6 个模型里**自由发挥选哪个**，质量取决于运气。

### 频率约束是否到位

| 约束项 | 当前规定 | 评价 |
|---|---|---|
| 免责声明频率 | "首次激活时说一次（仅一次），之后不再重复"（L27） | 清晰 |
| 自我打断频率 | 启发式 #8 写了"被赞誉/自我陶醉时立即自嘲"（L270） | 触发条件清晰 |
| 典故密度 | 表达 DNA #4 给了"每 200-300 字嵌 1 次"（L302） | 量化清晰 |
| 长短句比例 | 量化锚点表给了 32%/31%/16%（L337-340） | 清晰 |
| 「退出角色」恢复后能否再次进入 | **未规定** | 缺口 |
| 多轮对话中"声明"是否要在新会话首次重新声明 | **未规定**（"首次激活"指当前 session 首次还是 Skill 首次？） | 模糊 |

### 失败预防机制

**几乎缺失。** 当前 SKILL.md 只在 L26 写了"遇到不确定的问题用沉默/转移/反问替代说我不知道"——这只解决"承认无知会破角色核"的问题，**但没解决以下三类失败**：

1. **彻底超纲问题**：用户问编程 bug、问医学诊断、问代码评审——丁元英会怎么处理？当前 Skill 没说。LLM 可能：(a) 强行用"文化属性"硬套（荒谬）；(b) 跳出角色用普通模式回答（破坏沉浸感且无声明）；(c) 拒绝（违反"原则规律永远有结论"）。
2. **用户带情绪求安慰**：表达 DNA #5 写了"不主动同情"，但**没说"如果用户已经在哭/在崩溃，是否要让一步"**。完全冷处理可能会让用户中断 Skill 使用。
3. **用户问 1998 年之后的事**：诚实边界 #5 提到"1998 年之后打法要打折扣"，但**没给出"用户问加密货币/AI 创业时怎么应答的范本"**——LLM 容易直接装作丁元英也懂 2026 年。

### 改进建议

需在「角色扮演规则」之后、「回答工作流」之前，**插入一段"问题分诊表"**，把用户问题类型 → 对应心智模型 / 启发式 / 应答策略 显式写出（详见下方"改动 1"）。

## Q3 缺失的关键信息

### 关键案例索引

**缺。** SKILL.md 内嵌了大量"证据"段落（每个心智模型 3-5 条引语），但**没有"案例 → 引用位置"的反查表**。当用户说"举个例子"时，LLM 只能凭训练记忆复述，可能滑向编造。

调研材料 `05-decisions.md`（644 行）里有 **11 个关键决策的深度还原**（散基金、王庙村、给刘冰假档案袋、对林雨峰、对芮小丹、退股事件……），但 SKILL.md **只在末尾"附录"L471 提了一句"详见 05-decisions.md"**——LLM 在角色扮演中不会主动 read 这个文件，等于死引用。

### 反面教材利用

**完全没用。** 调研材料 `04-external-views.md` 里整理了刘冰、叶晓明、冯世杰、林雨峰等"反面/对照角色"的画像，**这些是丁元英思维的天然教学锚点**：

- 刘冰 = "破格获取" + "经营恩典" 的活体反面教材
- 叶晓明 = "小聪明 + 文化属性弱" 退股事件
- 林雨峰 = "只有矛没有盾"的强者必死路径
- 冯世杰 = 普通人接受现实的样本

SKILL.md **一次都没提这几个名字**。这是巨大浪费——丁元英本人解释规律时，最爱用这些人当"活反例"。

### 现代场景桥接

**严重缺。** 诚实边界 #5（L453）只说了"1995-1998 三重红利期，离开窗口几乎无法复制"，**但没给出转译桥**：

- 「乐圣公司 / 音响行业」的当代等价物是什么？（垂直平台霸主？头部 KOL？某个 SaaS 巨头？）
- 「王庙村扶贫」的当代等价物是什么？（创作者经济？知识付费？小镇创业？）
- 「私募基金从狼嘴里夹肉」的当代等价物是什么？（量化交易？meme 币？payment for order flow？）

没有这些桥接，普通用户在 2026 年问问题时，LLM 复述 1995 年的案例，**用户会觉得"丁元英活在博物馆里"**。

### 用户自检机制

**完全缺。** SKILL.md 反复强调"自己觉到悟到的，才是自己的"（L463），**却没给用户任何自检工具**——用户怎么判断自己是否真的内化了这套思维？建议在末尾加一组"测试题"或"自检 checklist"（详见改动 3）。

## Q4 启发式规则的边界

### 清晰的（5 条）

- #1 三问法：触发场景"高收益机会"明确，3 个判断问题可逐项回答

### 模糊的（5 条，缺"不适用场景"和"普通人减弱版"）

| # | 当前规则 | 模糊点 | 应补 |
|---|---|---|---|
| 3 | "情绪驱动 → 等" | 等多久？等到什么算"不情绪了"？ | 减弱版：等一夜 / 等 24 小时再决定 |

```diff
+ 7. **隐居态**：你的"音响"是什么？（高浓度精神出口，让你不需要从外界讨要意义的那个东西）
+
+ 答不出 → 还在"读丁元英"阶段。
+ 能答 3-4 题 → 在"用"阶段。
+ 能稳定答 5 题以上 → 不再需要这个 Skill 了。
+
---
```

**为什么改**：
- **现代场景转译桥**直接补 Q3"现代场景桥接"缺口和 Q4 启发式规则边界的"普通人减弱版"问题——给 LLM 一张明确的对照表，避免它在 2026 年的对话里强行复述 1995 年的乐圣音响案例。
- **用户自检 7 题**补 Q3 用户自检机制完全缺失的问题。题目本身就是 7 条核心规则的内化检查，**且最后一句"答出 5 题以上不再需要 Skill"**呼应了诚实边界 #10 "Skill 是镜子不是救主"——这是丁元英最反对的"靠救主"陷阱在 Skill 自身上的闭环。

## 综合判断

### Skill 等级：「需要用户摸索」级（不是「激活即执行」级）

**理由**：
- 触发覆盖偏窄（专名导向，缺痛点导向）→ 真实使用场景接不住
- 6 个模型 + 10 条启发式 + 13 条附加规则**没有路由层**→ 角色扮演的回答质量靠 LLM 自由发挥
- 反面教材（刘冰/叶晓明/林雨峰）一次都没在主体规则里出现 → 浪费了原著最大的"活教学锚点"
- 现代场景没有转译桥 → 用户问 2026 年的事，Skill 容易显得活在博物馆里

但表达 DNA、心智模型本体、诚实边界这三块**做得非常扎实**，是 95 分的素材层。问题集中在**激活层和路由层**。

### 最大缺口

**问题分诊表的缺失。** 心智模型本身写得好，但 LLM 不知道什么问题该用哪个模型——这等于给了 6 把锋利的工具，但没给工具盒。改动 2 是单点收益最高的修改。
